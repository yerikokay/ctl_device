## v0.0.6 ChangeLog

### 亮点

#### 启动架构简化
- **默认 full 模式**：直接 `./ctl_device` 启动，无需 `server` 子命令
- **`--connect` 自动变 client**：`./ctl_device --connect http://x.x.x.x:3711` 自动注册 agent、发送心跳
- **统一配置文件**：`mode / connect / server / client / notify / projects` 合并为一个 `config.yaml`
- **平级子命令**：`mcp / status / dispatch / logs` 不再嵌套在 `client` 下
- **向后兼容**：`server` 和 `client` 子命令保留，打印 deprecated 提示后正常运行

#### gRPC Server（:3713）
- **AgentService**：Register / Heartbeat / ListAgents
- **ProjectService**：Register / List
- **TaskService**：Get / UpdateStatus / Complete / Block / Dispatch / Advance
- **EventService**：Subscribe（server-streaming 实时事件流）
- Token 认证：gRPC metadata interceptor（`Authorization: Bearer <token>`）
- 完整单元测试覆盖（auth / no-auth / 各 service）

#### Full 模式唯一性锁
- 启动时创建 PID lock file（`<state_dir>/ctl_device.lock`）
- 检测到同 PID 活跃时报错：`Another full instance is already running (PID: xxx). Use --connect to join as client`
- 进程异常退出后 stale lock 自动清理

### 修复
- CI/CD Build matrix YAML 格式错误（`os` 字段含逗号导致解析失败）
- `--grpc-port` CLI flag 补充

### CLI 变更
```bash
# v0.0.5（旧）
ctl_device server
ctl_device server --token xxx
ctl_device client mcp --server http://x:3711

# v0.0.6（新）
ctl_device
ctl_device --token xxx
ctl_device mcp --server http://x:3711
ctl_device --connect http://x:3711   # client 模式
```
